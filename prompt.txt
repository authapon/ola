จาก code ที่ให้ไป จงวิเคราะห์ code เหล่านี้อย่างละเอียด รอบคอบ ในส่วนของ system prompt ของ coding subcommand นั้น มีการวางแผนพัฒนา coding ด้วยหลักเกณฑ์อะไร ถ้าหากให้วางแผนพัฒนาในลักษณะเป็นราย feature หรือ ราย function แล้วการตรวจสอบหรือแก้ไขข้อผิดพลาดน่าจะดีกว่าไหม หรือควรวางแผนด้วยหลักเกณฑ์แบบไหน ช่วยแนะนำให้ด้วย รวมทั้งส่วน vibe coding ของ ask subcommand ด้วย อีกอย่างครับ ถ้าต้องการเพิ่มการใช้ tool calls สำหรับการค้นหาข้อมูลเพิ่มเติมจาก internet ด้วย playwright และ searxng บน local โดยสามารถ config ผ่าน environment variables และ flags และให้สามารถค้นหาข้อมูลและโหลดข้อมูลจากเวบแบบขนานได้ด้วยเพื่อความเร็ว มีคำแนะนำอย่างไรกับส่วนนี้ ช่วยแนะนำเพิ่มในส่วนนี้ด้วยครับ


ดำเนินการตามที่แนะนำได้เลยครับ แล้วอีกอย่างครับ ในกรณีใช้งาน ask subcommand ถ้าหากไม่ใช้งาน vibe coding ให้อย่าทำการ verify หรือ build อะไรอีก ให้ตรวจสอบงานที่ทำว่าเป็นงานแก้ code หรือ งานแก้เอกสาร หรืองานทั่วไป เพราะผมเจอปัญหาว่า พอให้แก้ดอกสาร .txt แล้ว ระบบพยายามจะ build และ verify ด้วย nodejs ช่วยปรับปรุงส่วนนี้ด้วยครับ และอีกอย่างคือ ช่วยเพิ่มการใส่ prompts ผ่านไฟล์ด้วยครับ ใน ask subcommand และอีกอย่างคือ ช่วยเพิ่มการแจ้งเวลาที่ใช้ในการ preload ด้วยครับ นอกเหนือจาก thinking time
